# Results

This directory stores all outputs generated by the experiment pipeline. Files here are produced automatically by `src/main.py`, `src/evaluate_model_ood.py`, and `src/run_optuna.py` — do not edit them manually.

---

## Directory Layout

```
results/
│
├── MLP_Standard/            ← Per-model output folders
├── MLP_Sparse/
├── MLP_Dropout/
├── Polynomial/
├── PySR/
├── GPLearn/
├── PySINDy/
├── QLattice/
│
├── all_models/              ← Cross-model comparison outputs
│   ├── combined_results.txt ← MSE/MAE table for all models × all datasets
│   ├── gallery_residuals.png
│   └── gallery_loss.png
│
├── datasets_plots/          ← Visualisations of the raw datasets
├── optuna_hyperparams/      ← Optuna study outputs (best params, importances)
├── results_ood/             ← Out-of-distribution evaluation metrics and plots
└── weights/                 ← Serialised model weights for OOD reuse
```

---

## Per-Model Folders

Each model folder (e.g., `results/PySR/`) contains one subdirectory per dataset run:

```
PySR/
└── kepler_no_noise/
    ├── metrics.json           ← {"mse": ..., "mae": ..., "equation": "..."}
    ├── residual_analysis.png  ← Residual scatter plot (300 DPI, LaTeX fonts)
    └── loss_curve.png         ← Training / validation loss 
```

All metrics are computed in the **original physical space** (after inverse-transforming standardised predictions).

---

## Cross-Model Outputs (`all_models/`)

| File | Description |
|---|---|
| `combined_results.txt` | Full MSE / MAE table across all models and datasets |
| `gallery_residuals.png` | Grid of residual plots — one panel per model × dataset |
| `gallery_loss.png` | Grid of loss curves — MLP models only |

Generated automatically at the end of `python src/main.py`.

---

## OOD Results (`results_ood/`)

Contains metrics and plots produced by `src/evaluate_model_ood.py` for inputs **outside the training domain**. File naming mirrors the structure of per-model folders, with an `_ood` suffix.

---

## Optuna Outputs (`optuna_hyperparams/`)

Contains the outputs of `src/run_optuna.py` and `aggregate_hyperparams.py`:

- `optuna_<model>_<dataset>.txt` — Pareto-optimal hyperparameter configurations.
- `PROMEDIO_HIPERPARAMETROS.txt` — Cross-model aggregate hyperparameter from Optuna .

---

## Weights (`weights/`)

Copy of model states saved by `save_model_weights()` in `src/utils/io.py`.

---

## Plot Standards

All figures follow a consistent academic style enforced by `src/utils/utils.py`:

- **Font:** Latin Modern Roman (LaTeX rendering via `matplotlib`).
- **Resolution:** 300 DPI.
- **Format:** PNG.
